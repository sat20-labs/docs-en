# Builder Program

Builder Program attracts and supports SatoshiNet ecosystem builders. It serves long-term public goods, real applications, and verifiable infrastructure, not short-term campaigns.

## Current Priority Builders

| Direction | What We Want to See |
| --- | --- |
| BTC community pilots | Communities willing to deploy community DEX, DAO, Explorer, and wallet entry on testnet |
| EVM developers | Teams that can bring AMM, limit orders, Launchpad, payment, stablecoin, and other contracts to SatoshiNet |
| Infrastructure teams | Core Node, L1/L2 Indexer, Explorer, RPC, monitoring, deployment tools |
| Wallets / exchanges | Bitcoin L1 and SatoshiNet deposits/withdrawals, asset query, STP state, authorization flow |
| AI Agent developers | Agent Wallet & Safety, Community Builder Agent, contract assistants, deployment assistants |
| Liquidity partners | AMM pools, limit order markets, cross-community liquidity, market-making tools |
| Security and audit | STP, contracts, wallet adapter, Indexer, cross-layer state validation |
| Content and regional communities | Tutorials, videos, translations, community operations, local developer support |

## What Builders Can Create and Receive

| Builder | Value Created and Received |
| --- | --- |
| BTC community | Own brand, user entry, application rules, community relationships; can design application-layer service fees |
| Developer | Application revenue, deployment and integration revenue, public case studies, possible future grants |
| Mining Node | GAS fees generated by actual block production and execution; other rewards follow final protocol rules |
| Core Node | Includes Mining Node capability and provides STP service; can receive block-production fees, while extra STP service fee model remains under design |
| Indexer / RPC / Explorer | API, SLA, hosting, and enterprise integration revenue |
| Liquidity partner | Fees explicitly generated by AMM, limit order, or application protocols, while bearing market risk |
| Content and regional community | Reputation, cooperation projects, and future ecosystem service opportunities |

Current Builder Program mainly provides open-source code, testnet, technical review, deployment support, and joint showcase. Cash grants are not guaranteed; they may later come from dedicated sponsorship, grants, or foundation programs.

## How to Apply

Application material should include project goal, team background, target users or community, required modules, current status, first testnet milestone, existing code or evidence, risk boundaries, and questions requiring SAT20 Labs review.
