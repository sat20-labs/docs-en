# Builder Program

The Builder Program attracts and supports SatoshiNet ecosystem builders. It serves long-term public goods rather than short-term activities.

## Support Areas

| Area | Examples |
| --- | --- |
| Infrastructure | Wallets, indexers, explorers, SDKs, monitoring |
| STP clients | Multi-language adapters, PWA bridge, testing tools |
| Smart contracts | AMMs, stablecoins, payments, asset markets |
| AI Agents | SAT20 Agent Wallet skill, wallet assistants, contract assistants, safety verification |
| Education | Tutorials, videos, reproducible testnet drills |
| Research | RSMC, GAS economics, contract safety, cross-layer state |

## Evaluation Criteria

1. Does it strengthen user asset control?
2. Does it lower developer integration cost?
3. Does it improve SatoshiNet asset liquidity?
4. Does it create reusable public goods?
5. Does it provide testnet or mainnet verifiable results?

## Project Information

Each ecosystem project profile contains:

1. Project goal.
2. Current state.
3. Usage.
4. Risk boundary.
5. Testnet or mainnet evidence.
6. Relationship with STP, GAS, contracts, or Agents.
7. Whether it provides verifiable asset facts, transaction evidence, or state proofs.
