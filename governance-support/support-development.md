# Support Protocol Development

Long-term protocol development needs transparent and sustainable support. Supporting protocol development does not equal buying GAS, and does not represent price upside, network profit, or governance-right promises.

## Support Methods

| Method | Use |
| --- | --- |
| BTC donation | Support open-source maintenance, docs, testnet, and public services |
| Open-source grants | Support clearly scoped public-good milestones |
| Enterprise sponsorship | Support EVM Runtime, security audits, Wallet SDK, public Indexer / Explorer / RPC |
| Technical services | Community Stack deployment, wallet or exchange integration, hosted node and API services |
| Infrastructure resources | Servers, bandwidth, monitoring, nodes, and explorers |
| Documentation and translation | Support Chinese, English, and regional community materials |

## Boundary Statement

1. Supporting protocol development is not buying GAS.
2. Supporters are not promised price returns or network profit.
3. Before a future foundation is established, funds are not received or managed under a foundation name.
4. A formal funding page should list the receiving entity, use of funds, budget, responsible person, and deliverables.
