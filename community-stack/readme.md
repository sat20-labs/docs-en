# Community Stack: Build Complete Infrastructure for Your BTC Community

Community Stack is SatoshiNet's complete infrastructure package for BTC communities. Its goal is not only to let communities "bridge" assets to another network, but to help them gradually own their DEX, DAO, wallet entry, Indexer, Explorer, Launchpad, AMM, limit order system, and operations backend.

> Let every BTC community own its own financial infrastructure.

## What a Community Can Own

| Module | Role |
| --- | --- |
| Core Node | Provides SatoshiNet node capability, STP service, and community base service |
| L1 / L2 Indexer | Expresses Bitcoin L1 and SatoshiNet L2 asset facts |
| Explorer | Shows transactions, assets, contracts, channels, and cross-layer evidence |
| Wallet / Wallet SDK | Lets users hold assets, sign, authorize, and verify state |
| Launchpad | Supports community asset issuance, events, and initial distribution |
| AMM | Supports automated market making and base liquidity |
| Limit order system | Supports order-book trading and professional market making |
| Transcend / STP | Supports Bitcoin L1 asset entry and exit |
| DAO | Supports governance, parameter management, and community fund workflows |
| DEX frontend and backend | Supports the community's own trading entry and operations management |
| AI Agent | Supports requirement analysis, deployment config, safety checks, and operations reports |

## Deployment Modes

| Mode | Content | Current Status |
| --- | --- | --- |
| Open-source self-hosting | Community deploys code, nodes, and services by itself | Planning |
| Assisted deployment | SAT20 Labs or ecosystem service providers provide architecture, deployment, integration, and training | Planning |
| Hosted operations | Nodes, indexers, explorers, monitoring, and upgrades as service | Planning |
| Strategic co-building | Deep customization, joint testing, liquidity, and ecosystem cooperation | Planning |

SAT20 Labs' goal is not to make every community depend on us, but to let each community gain the ability to run its own infrastructure independently.

## Community Requirements Form

When submitting a community cooperation request, include:

1. Community name and contact.
2. Website, X, Telegram / Discord.
3. Asset protocol, ticker, holders, and current liquidity.
4. Required modules: nodes, Indexer, Explorer, wallet, DEX, DAO, Launchpad, AMM, limit orders, Agent.
5. Whether the community can run nodes, Indexer, and Explorer itself.
6. Liquidity plan.
7. DAO and signer rules.
8. Testnet launch goal.
9. Budget or resource capacity.
10. What support is expected from SAT20 Labs.
