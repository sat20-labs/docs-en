# Why Bitcoin Needs an Open Execution Layer

Bitcoin L1 prioritizes simplicity, verifiability, scarce assets, and final settlement. It is not designed to carry every high-frequency trade, complex contract, community governance flow, or application execution. As Bitcoin-native assets, Ordinals, Runes, BRC20, ORDX, and more application demand appear, execution naturally moves to other layers.

The real question is not whether execution needs to scale. The real questions are:

1. Whether the execution layer can still trace Bitcoin L1 asset facts.
2. Whether users retain exit and protection paths in abnormal situations.
3. Whether communities can run their own infrastructure independently.
4. Whether developers can build applications on top of Bitcoin-native assets.
5. Whether the network avoids returning users to centralized custody.

SatoshiNet is one open implementation of this path. It is not trying to replace Bitcoin. It gives Bitcoin-native assets an execution environment better suited for applications.

## SatoshiNet's Path

1. Bitcoin L1 provides asset origin and final boundaries.
2. Indexer provides queryable and verifiable asset facts.
3. STP provides cross-layer control, exit, and punishment paths for revoked states.
4. SatoshiNet provides a faster, lower-cost, programmable execution environment.
5. Wallets and AI Agents turn complex protocol mechanics into verifiable, authorized user operations.

## Where ORDX Fits

ORDX is the Satoshi-denominated asset protocol in the SAT20 stack. Its history and protocol value belong in [Asset Issuance Protocols](protocol/indexer/asset-issuance.md), not as the only site-wide narrative. SatoshiNet is broader: it covers multi-protocol Bitcoin L1 assets, cross-layer control, open execution, node networks, and community-owned infrastructure.
